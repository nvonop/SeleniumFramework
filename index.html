Welcome to Selenium Framework

Introduction

This framework is intended to provide a simple way of writing tests in Selenium by wrapping common Selenium features in an easier to use package.

The goal is to abstract away the lower level Selenium API so that the less technical users who may want to write tests can do so.

The documentation here is constantly evolving. If you have questions that these pages don't answer then please ask. If you wish to help out volunteers are always welcome.

Why Use The Framework?

It is lightweight and hopefully makes writing tests easier, leaving you to focus on the automation. The frameowrk provides full access to the underlying Selenium API for all objects it wraps so if the framework hasn't implemented something you can still work with Selenium directly.

It helps with alot of synchronisation issues you might find when trying to automate a web application.

It is easy to run your tests against a different web browser with a single line change. In fact it is easy to point your tests at a completely different URL (test environment), control the speed of the tests, or to choose to run them remotely, on your local machine, or on Sauce Labs, with single parameter changes. This is done through the use of system properties which are set when the tests are run. I would recommend running the tests through a build tool such as Jenkins and/or Gradle and passing the parameters into the test, rather than having them hard coded.

The framework also enables you to take screenshots when your tests are running and store these for later analysis.
