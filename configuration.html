How to Configure the Framework

The framework is dependent on several system properties that have to be set before tests can be run. These can either be set within a build file (see this projects build.gradle) or within a setup method provided by either JUnit, TestNG or Spock.

The list of properties that must be set are as follows:

LOCAL_DRIVER - A boolean value indicating whether or not the driver is running locally.

REMOTE_DRIVER - A boolean value indicating whether or not the driver is running on a grid.

SAUCE_DRIVER - A boolean value indicating whether or not the driver is running on Sauce Labs.

SAUCE_KEY - A String representing the sauce key. Required if SAUCE_DRIVER is set to true.

GRID_HOST - A String representing the URL for the selenium grid.

GRID_PORT - A String representing the port number for the selenium grid.

BROWSER - A String representing the browser that the tests will be run on. "Firefox", "Internet Explorer" and "Chrome" are supported.

VERSION - A String representing the browser version number. This is only required if REMOTE_DRIVER or SAUCE_DRIVER are set to true.

PLATFORM - A String representing the platform the tests will run on. This is only required if REMOTE_DRIVER or SAUCE_DRIVER are set to true.

TIMEOUT - A String representing the duration the framework will search for an element before throwing a TimeoutException

DELAY - A String representing the "pause" time before carrying out an action such as a click

IGNORE_SECURITY_DOMAINS - A boolean defining the value of the InternetExplorerDriver.INTRODUCE_FLAKINESS_BY_IGNORING_SECURITY_DOMAINS property within the IEDriver.

TEST_RESULTS_PATH - A String representing the location where test evidence can be stored. Screenshots taken by the framework will be stored here. The path can be modified within the tests however.

APPLICATION_URL - A String representing the URL for the application.
