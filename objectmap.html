The Object Map

This class enables By locators to be retrieved from a properties file. This enables locator information to be stored externally to the tests. This is taken from a recipe in The Selenium Cookbook (a highly recommended purchase!).

The following methods are provided:

  • getLocatorFromMap(String logicalElementName) - returns a By object used to locate a WebElement.
  • getLocatorLocatorAndOverride(String logicalElementName, String locatorValue) - returns a By object used to locate a WebElement. This method can be used to override locator details stored within an object map. Useful for cases where a locator id (or other attribute) is dynamic.
  • getLocatorString(String logicalElementName) - returns the type of locator stored in the map.
  • getLocatorValue(String logicalElementName) - returns the value of the locator string.

It can be used as follows:

private ObjectMap map = new ObjectMap("ControlsPageObjectMap.properties");
public Button button1 = new Button(driver, map.getLocator("button_1"));

Note: The Button object is from the Controls package. Refer to ControlsPage.java within the Tests package for further details of how to use this class.

The Properties File

The properties files read by the ObjectMap class are in a defined format. This format is as follows:
PROPERTY_NAME=LOCATOR_TYPE>LOCATOR_STRING

In the code example above, we referenced ControlsPageObjectMap.properties:
button_1_click_message=id>pg1
button_2_click_message=id>pg2
radio_button_1_click_message=id>pg3
radio_button_2_click_message=id>pg4
checkbox_1_click_message=id>pg5
checkbox_2_click_message=id>pg6
iframe_1=name>iframe_1
button_1=name>button1
button_2=id>btn2
alert_button=css>input#btn3
button_4=xpath>//*[@id=\"btn4\"]
radio_button_1=name>sex
radio_button_2=id>rb2
checkbox_1=css>input#cb1
checkbox_2=xpath>//*[@id=\"cb2\"]
link_google=id>lnk1
table_1=name>table1
text_firstname=css>input#txt1
text_lastname=xpath>//*[@id=\"txt2\"]
select_cars=id>ddl1
button_submit=id>sbmt
